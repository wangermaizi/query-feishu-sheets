# 更新日志

本文件记录项目每个发布版本中面向使用者的重要变化。

## [v0.3.0] - 2026-07-21

### 新增

- 新增手动需求模式，没有飞书需求表时可直接描述一条或多条需求，自动生成本地需求 ID，并复用完整分析、确认、实施与群推送流程。
- 新增 `fast`、`standard`、`strict` 三级复杂度策略，按改动范围和风险调整测试、Reviewer 数量及复审轮次。
- 新增仓库与 worktree 级并行隔离，不同项目可并行处理，同一工作目录保持冲突保护，相同需求 ID 不再跨项目覆盖。
- 新增分支命名规范 `<type>/<YYYYMMDD>/<summary>`，支持新增、修复和重构类型并在创建前确认完整名称。

### 改进

- 首次分析同时输出原始需求、普通语言需求说明和技术结论，说明现状、用户影响、期望结果及必要示例，并在任务恢复和飞书结果卡片中保留。
- Review 改为强制阶段门禁：主 Agent 完成全部实现和初测后才允许 Reviewer 审查同一份完整候选，收齐结论后集中修复，避免边改边审和无效重复复审。
- 状态文件增加进程锁、仓库范围迁移和旧活动记录关联能力，不再要求通过跳过其他项目需求解除全局锁。
- 最终报告增加复杂度、Review 策略和审查时间线校验，防止实现前 Review 或超出规定轮次的结果被发布为成功报告。

## [v0.2.1] - 2026-07-21

### 改进

- 精简首次配置说明，区分最小配置和一次性完整配置，并明确字段映射、群 ID 与模型参数由 Codex 自动发现或按需询问。

## [v0.2.0] - 2026-07-21

### 新增

- 新增需求实施必要性核验，区分尚未开始、部分完成、已完成、失效、重复和需要人工判断，并记录可审计证据。
- 新增多需求表语义匹配和默认 profile 切换，支持显示名、别名、用途描述及多候选人工选择。
- 新增整批需求统一确认流程，确认后按优先级每轮实施一条，其余批准项留待后续处理。
- 新增首次配置筛选条件确认和空筛选保护，只有明确授权 `allow_unfiltered` 才允许整表查询。
- 飞书结果卡片标题增加项目名和需求 ID，格式为 `【项目名｜需求ID】需求标题`。

### 改进

- 构建脚本在测试或 Skill 校验失败时立即停止，避免生成无效产物。

### 修复

- 为 GitHub Release job 显式设置目标仓库，修复未 checkout 时 `gh release` 无法定位仓库导致的发布失败。

## [v0.1.0] - 2026-07-17

### 新增

- 新增飞书需求自动执行 Skill，支持读取 Sheets 和 Bitable、分析排序并每轮选择一条需求。
- 新增代码修改前的分支确认，以及主 Agent 实施、三个只读 Agent 独立审查和统一修正流程。
- 新增本地处理状态、固定飞书群自动发送、结果哈希校验和敏感凭证保护机制。
- 新增离线安装包、SHA-256 校验文件及 GitHub Actions 自动构建和标签发布流程。
- 新增仓库发布 Skill，可自动整理更新日志、创建语义化版本并推送到 GitHub。

### 改进

- 流水线启动检测优先使用 GitHub CLI，并在本机未安装 CLI 时支持通过 GitHub REST API 查询公开仓库。
